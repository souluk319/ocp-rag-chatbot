### 2. Qwen3.5 thinking 모드 호환성 문제 해결

**문제:**
- RTX Desktop(Ollama), Mac Mini(MLX) 모두 Qwen3.5의 thinking 모드가 기본 활성화
- 응답 구조: `content: ""` (빈 문자열) + `reasoning: "Thinking Process: ..."` (영어)
- 회사 서버(vLLM)는 `chat_template_kwargs: {"enable_thinking": false}`로 정상 동작

**진단 과정:**
1. RTX Desktop: 답변은 빠르게 나오지만 영어로만 출력됨
2. Mac Mini: 로딩 애니메이션만 돌고 답변이 안 나옴 (content가 빈 문자열이라 화면에 표시 안 됨)
3. 직접 curl로 각 서버 응답 확인 → 둘 다 `content: ""`, `reasoning: "..."` 동일 증상

**해결:**
- 각 백엔드의 thinking 비활성화 파라미터가 다름:
  - vLLM: `chat_template_kwargs: {"enable_thinking": false}`
  - Ollama: `reasoning_effort: "none"`
  - MLX: 서버 측에서 thinking 비활성화 설정 필요
- 해결 방법: payload에 **두 파라미터 모두** 전송 → 각 백엔드가 자기가 이해하는 파라미터만 사용하고 나머지는 무시
- 안전장치: `content`가 비고 `reasoning`이 있으면 `reasoning`을 fallback으로 사용

### 3. 모델명 자동 감지

**문제:**
- config에 `Qwen/Qwen3.5-9B`로 설정되어 있지만 Ollama는 `qwen3.5:9b`, MLX는 `mlx-community/Qwen3.5-9B-4bit` 사용
- 모델명 불일치로 HTTP 에러 발생

**해결:**
- `auto_detect_model()` 메서드: 엔드포인트 전환 시 `/v1/models`로 실제 모델명 자동 감지
- RTX → `qwen3.5:9b`, Mac Mini → `mlx-community/Qwen3.5-9B-4bit` 자동 설정
