챗봇 품질 개선 (프론트엔드, 검색, 캐시, 세션) + 코퍼스 보강 + UI 고도화 + 듀얼 모드 시스템

## 완료한 작업

### 1. 보안 정리

- `data/index/` git 추적 제거 → `.gitignore`에 추가 (clone 후 `make index`로 재생성)
- 보안 사고 관련 문서 2건 삭제 (`docs/github-publication-security-plan-2026-03-22.md`, `docs/publication-security-remediation-plan-2026-03-22.md`)
- `CLAUDE.md` 생성 — 프로젝트 보안 지침 강화 (IP/도메인/시크릿 하드코딩 금지, 커밋 전 체크리스트)
- `src/config.py` — 잘못된 기본 포트 수정 (`localhost:8081`, `localhost:8082` → `localhost:8080`)

### 2. 마크다운 테이블 렌더링

**문제:** 챗봇 응답에 마크다운 표(`| --- |` 형식)가 그대로 텍스트로 출력됨

**해결 (`frontend/index.html`):**
- `renderMarkdownTable()` 함수 추가: `|---|---|` 패턴 감지 → HTML `<table>` 변환
- `renderMarkdown()` 수정: 테이블을 먼저 추출한 후 나머지 마크다운 규칙 적용
- CSS 추가: `.message table`, `.message th`, `.message td` 스타일

### 3. 검색 결과 소스 중복 제거

**문제:** 검색 결과 5개 중 같은 문서(md 파일)에서 3개 이상 나오는 경우 발생 → 출처 다양성 부족

**해결 (`src/retriever/__init__.py`):**
- Reranking 후 `max_per_source = 2` 적용: 같은 source에서 최대 2개까지만 선택
- `source_count` 딕셔너리로 문서별 카운트 관리
- 간단하지만 효과적 — 복잡한 MMR 알고리즘 대신 심플한 카운트 제한

### 4. 캐시 숫자 인식 개선

**문제:** "세줄요약"과 "네줄요약"이 코사인 유사도 0.95+ → 같은 캐시 응답 반환

**해결 (`src/cache/__init__.py`):**
- `_extract_numbers()` 메서드 추가: 쿼리에서 숫자 추출
- `_is_cache_compatible()` 수정: 숫자가 다르면 캐시 미스 처리
- 기존 토큰/엔티티 가드와 함께 3중 호환성 검사 (숫자 → 엔티티 → 토큰 교집합)

### 5. 타이핑 인디케이터 파이프라인 상태 표시

**변경 (`frontend/index.html`):**
- `STEP_STATUS` 매핑 추가: 파이프라인 각 단계별 한국어 상태 텍스트
- `updateTypingStatus()` 함수: 타이핑 애니메이션 아래에 현재 단계 텍스트 표시
- `.typing-status` CSS 클래스 추가

### 6. Query Rewrite 프롬프트 개선

**문제:** "그거 삭제하려면?" → "생성 방법..."으로 재작성됨 (동사가 바뀜)

**해결 (`src/session/__init__.py`):**
- REWRITE_PROMPT를 few-shot 예시 기반으로 전면 재작성
- 3개 구체적 예시 추가 (대명사 → 구체적 단어, 동사 보존)
- 규칙 간소화: 3줄로 핵심만

### 7. 코퍼스 보강

**Phase 1: 공식문서 스크래핑 (`scripts/scrape_docs.py`)**
- OCP URL 8개 + K8s PDB URL 1개 추가
- 404 오류 7건 발생 → Red Hat 4.17 URL 구조 변경 확인 → URL 수정 후 재스크래핑 완료

**Phase 2: 한국어 합성 문서 (`scripts/generate_docs.py`)**
- 토픽 6개 추가 (인덱스 8~13): Namespace/Project, Node 관리, 업그레이드, Route/Ingress, 빌드/배포, 로깅

### 8. 타이핑 인디케이터 버그 수정

**문제:** "답변 생성 중" 텍스트가 dots 애니메이션과 겹쳐서 UI 깨짐

**원인:** `.typing-indicator span` 셀렉터가 `.typing-status` span까지 매칭 → width:8px, bounce 애니메이션 적용됨

**해결 (`frontend/index.html`):**
- 셀렉터를 `.typing-indicator span:not(.typing-status)`로 변경

### 9. 프론트엔드 디자인 폴리시

**변경 (`frontend/index.html`):**
- 커스텀 스크롤바 (6px, 다크 테마)
- 메시지 등장 fadeInUp 애니메이션
- 입력창 포커스 글로우 (보라색 box-shadow) + Send 버튼 클릭 피드백 (scale)
- 헤더/입력영역 box-shadow로 레이어 분리감
- 어시스턴트 메시지 호버 시 테두리 밝아지는 효과
- 유저 메시지 보라 그라데이션 (linear-gradient)
- 헤더 버튼 트랜지션
- 웰컴 화면 추천 질문 칩 3개 추가

### 10. 듀얼 모드 시스템 (학습/운영)

**변경:**
- `src/api/__init__.py` — `ChatRequest`에 `mode` 필드 추가 (`"learn"` / `"ops"`, 기본값 `"ops"`)
- `src/pipeline.py`:
  - 기존 `SYSTEM_PROMPT` → `SYSTEM_PROMPT_OPS` + `SYSTEM_PROMPT_LEARN`으로 분리
  - 학습 모드: 비유/예시 적극 활용, 단계별 설명, 기술 용어에 괄호 설명 덧붙임
  - 운영 모드: 기존과 동일 (간결, 실용, 명령어 중심)
  - `get_system_prompt(mode)` 함수로 모드별 프롬프트 선택
  - 캐시 키에 `[mode]` 접두사 추가 → 모드 간 캐시 간섭 방지
- `frontend/index.html`:
  - 입력창 왼쪽에 모드 셀렉터 드롭다운 (🔧 운영 / 📖 학습)
  - 모드별 추천 질문 풀 (각 8개) → 랜덤 3개 표시
  - `renderWelcome()` 함수로 동적 웰컴 화면 생성
  - 모드 전환 시 웰컴 화면 추천 질문 자동 갱신

### 11. 후속 추천 질문 (Perplexity 스타일)

**변경:**
- `src/pipeline.py` — 시스템 프롬프트에 `FOLLOWUP_SUFFIX` 추가: LLM이 `[추천 질문]` 섹션을 답변 끝에 출력하도록 지시
- `frontend/index.html`:
  - `extractFollowups()`: 답변에서 추천 질문 마커 파싱 → 본문/질문 분리
  - `appendFollowups()`: 추천 질문을 클릭 가능한 칩으로 렌더링
  - 스트리밍 중 추천 질문 섹션 이후 텍스트 숨김 처리
  - `done` 이벤트 시 최종 파싱 → 칩 표시
  - 칩 클릭 → 해당 질문 자동 전송
  - "이어서 질문해보세요" 라벨 추가
  - `.followup-suggestions`, `.followup-chip`, `.followup-label` CSS 추가
- **추천 질문 파서 유연화**: LLM이 `[추천 질문]` 외에도 다양한 형식으로 출력하는 경우 대응
  - 6가지 마커 패턴 매칭: `[추천 질문]`, `추천 질문:`, `관련 질문:`, `**추천 질문**`, `이어서 질문`, `다음 질문`
  - 불릿(`- `) 형식도 파싱
  - 너무 짧거나(5자↓) 긴(100자↑) 라인 필터링

### 12. 채팅 영역 "최신 대화" 점프 버튼

**변경 (`frontend/index.html`):**
- 스트리밍 중 스크롤을 위로 올리면 "↓ 최신 대화" 버튼이 하단 중앙에 페이드인 등장
- 클릭 시 맨 아래로 자동 스크롤 + 버튼 사라짐
- 기존 트레이스 패널의 "맨 아래로" 버튼과 동일한 UX 패턴
- `.chat-jump-btn` CSS + `updateChatJumpButton()` + `jumpChatToBottom()` JS 추가

## 발견한 문제 & 해결

| # | 문제 | 원인 | 해결 |
|---|------|------|------|
| 1 | 마크다운 표 렌더링 안 됨 | 프론트엔드에 테이블 파서 없음 | `renderMarkdownTable()` 추가 |
| 2 | 같은 문서가 검색 결과 독점 | 리랭킹에 다양성 제약 없음 | `max_per_source = 2` 제한 |
| 3 | "3줄" vs "4줄" 캐시 오탐 | 코사인 유사도만으로 판단 | 숫자 추출 비교 추가 |
| 4 | Query Rewrite 동사 변경 | 소형 LLM이 복잡한 프롬프트 이해 못함 | few-shot 예시 기반으로 변경 |
| 5 | "네임스페이스가 뭐야?" 답변 불가 | 코퍼스에 관련 문서 부재 | 코퍼스 보강 |
| 6 | config.py 기본 포트 오류 | 8081/8082 잘못 설정됨 | 전부 8080으로 수정 |
| 7 | 스크래핑 404 (7건) | Red Hat 4.17 URL 구조 변경 | URL 수정 후 재스크래핑 |
| 8 | 타이핑 상태 텍스트 겹침 | CSS 셀렉터가 `.typing-status`까지 매칭 | `:not(.typing-status)` 추가 |
| 9 | 추천 질문이 나올 때도 있고 안 나올 때도 있음 | LLM이 `[추천 질문]` 마커를 항상 동일한 형식으로 출력하지 않음 | 6가지 마커 패턴 매칭으로 파서 유연화 |
