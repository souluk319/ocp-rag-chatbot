# 2단계 - UI와 스트리밍 한글 복구

## 목표

UI 문구, 스트리밍 응답, 완료 후 표시 텍스트가 모두 정상 한글로 보이게 만든다.

## 왜 이 단계가 중요한가

기능이 살아 있어도 화면이 깨지면 심사에서는 실패로 보일 수 있다.

## 담당 역할

- Ramanujan: 어떤 지점에서 문자 깨짐이 생기는지 분류
- Arendt: UI 문구와 상태 문구를 자연스러운 한국어로 정리
- Peirce: 출력 문구가 공식 데이터 의미를 왜곡하지 않는지 점검
- Gibbs: SSE / TextDecoder / gateway 인코딩 경로 수정
- Feynman: 브라우저 기준 재현 테스트
- Russell: 정적 파일과 런타임 문자열 정합성 점검

## 작업 내용

1. 정적 HTML 파일의 한글 문자열을 점검한다.
2. gateway SSE 출력의 UTF-8 경로를 확인한다.
3. 브라우저 쪽 디코더와 남은 버퍼 처리까지 확인한다.
4. 상태 문구와 오류 문구를 한국어로 정리한다.

## 테스트

- 브라우저에서 `localhost:8000`
- 기본 질문 1건 스트리밍
- 완료 후 citation 표시

## 완료 기준

- 화면 한글이 깨지지 않는다.
- 스트리밍 응답 중 한글이 깨지지 않는다.
- 완료 후 상태 문구와 버튼 문구가 정상이다.
