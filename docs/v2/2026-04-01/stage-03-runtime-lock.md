# 3단계 - 로컬 런타임 경로 고정

## 목표

`localhost:8000` 이 항상 같은 방식으로 살아 있도록 로컬 런타임 경로를 안정화한다.

## 왜 이 단계가 필요한가

실행 때마다 포트, bridge, gateway 상태가 달라지면 테스트와 시연이 불가능하다.

## 담당 역할

- Ramanujan: 실행 흐름에서 꼭 필요한 경로와 불필요한 경로 구분
- Arendt: 사용자가 이해하기 쉬운 실행 문구 정리
- Peirce: active index / manifest 가 무엇을 가리켜야 하는지 점검
- Gibbs: bridge / OD / gateway 기동 경로 고정
- Feynman: 실제 프로세스 / 포트 / health 재확인
- Russell: README / 실행 문서와 실제 실행 경로 일치 여부 감사

## 작업 내용

1. 브리지, OD, gateway 의 실행 조건을 정리한다.
2. `.env` 와 런타임 계약을 검토한다.
3. `8000`, `18101`, `18102` 경로를 기준으로 상태를 안정화한다.
4. 로컬 실행 문서를 실제 상태와 맞춘다.

## 테스트

- `/health`
- `/`
- 간단한 `/api/v1/chat`

## 완료 기준

- `localhost:8000` 이 안정적으로 뜬다.
- health 가 정상이다.
- 브리지 / OD / 게이트웨이 관계가 문서와 실제가 일치한다.
