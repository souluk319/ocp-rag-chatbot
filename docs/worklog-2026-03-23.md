### 6. 하이브리드 리랭킹 근본 문제 해결 (핵심)

**증상:** eval에서 CrashLoopBackOff 검색 점수 0.732인데, 실제 챗봇에서 "데이터가 부족합니다" 응답

**원인 분석 (상세 → 트러블슈팅 문서 참조):**
1. 벡터 검색은 한국어 crashloop 문서를 1위로 찾음 (0.640)
2. 하이브리드 리랭킹에서 영문 문서의 BM25 점수(0.79~0.80)가 한국어 문서를 밀어냄
3. `max_per_source=2`로 영문 문서가 슬롯 독점 → 한국어 해결방법 문서가 top 5에서 탈락
4. LLM이 영문 에러 테이블 + 일반 개념 설명만 받고 "해결방법 데이터 부족"으로 자체 판단

**수정 4건:**

| 파일 | 변경 | 효과 |
|------|------|------|
| `src/retriever/__init__.py` | `max_per_source`: 2 → 1 | 영문 문서 슬롯 독점 방지 |
| `src/retriever/__init__.py` | 한국어 쿼리 시 `-ko.md` 문서 +0.08 부스트 | 한국어 문서 순위 보정 |
| `src/retriever/__init__.py` | `build_context max_chars`: 4000 → 6000 | 5위 문서도 컨텍스트에 포함 |
| `src/pipeline.py` | 시스템 프롬프트: "절대 답변하지 마세요" → "최대한 활용하여 답변하세요" | LLM이 가용 컨텍스트로 최선의 답변 생성 |

## 발견한 문제 & 해결

| # | 문제 | 원인 | 해결 |
|---|------|------|------|
| 1 | Windows 콘솔 UnicodeEncodeError | cp949 인코딩에서 em-dash 등 출력 불가 | ASCII 텍스트로 변경 / `PYTHONIOENCODING=utf-8` |
| 2 | Git author identity unknown | Windows에 git user config 미설정 | `git config --global user.name/email` |
| 3 | 서버 재시작해도 옛날 인덱스 사용 | `--reload`로는 IVFIndex 재로드 안 됨 | kill.bat → run.bat 풀 재시작 |
| 4 | 시맨틱 캐시 "못 찾겠다" 응답 반환 | 데이터 추가 전 캐시된 응답이 남아있음 | 서버 재시작으로 캐시 클리어 |
| 5 | n_probe config 변경이 반영 안 됨 | `IVFIndex.load()`가 meta.json 값 사용 | config의 `IVF_N_PROBE` 값 우선 적용 |
| 6 | CrashLoopBackOff 문서 검색 안 됨 | IVF 클러스터 누락 (n_probe 부족) | n_probe 6→16 (전수탐색) |
| 7 | 검색은 되는데 LLM이 "데이터 부족" 응답 | 하이브리드 리랭킹이 영문 문서 과대평가 | max_per_source=1, 한국어 부스트, 프롬프트 완화 |
| 8 | kill.bat 실행해도 Python 프로세스 안 죽음 | Claude 백그라운드 태스크가 Python 점유 | `taskkill /F /PID`로 특정 PID 종료 |
| 9 | 스크래핑된 파일이 빈 껍데기 (6건 <1KB) | Red Hat 문서가 JS SPA 렌더링 | Playwright 업그레이드 필요 (미착수) |
| 10 | `data/raw/` 디렉토리 Windows에 없음 | `.gitignore`에 포함되어 전송 안 됨 | `mkdir` + 문서 재생성 |
