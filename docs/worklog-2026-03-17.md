OCP RAG 챗봇 프로젝트 전체 파이프라인 구현 + 데이터 수집 + UI 완성

## 완료한 작업

### 1. 전체 RAG 파이프라인 구현
프레임워크(LangChain) 없이 모든 컴포넌트를 직접 구현

| 모듈 | 파일 | 설명 |
|------|------|------|
| Chunker | `src/chunker/__init__.py` | Sliding Window 청킹 + Overlap + 문서 파싱 (txt, md, pdf, docx, pptx) |
| Embedding | `src/embedding/__init__.py` | sentence-transformers + LRU 캐싱 (SHA256 키, 최대 5000개) |
| VectorStore | `src/vectorstore/__init__.py` | IVF 인덱스 직접 구현 (K-Means 클러스터링, Cosine Similarity) |
| Retriever | `src/retriever/__init__.py` | Hybrid Reranking (Semantic 70% + BM25 30%) |
| Session | `src/session/__init__.py` | 세션 관리 + LLM 기반 Query Rewriting (대명사 해소) |
| Cache | `src/cache/__init__.py` | Semantic Response Cache (cosine similarity 0.95 임계값) |
| LLM | `src/llm/__init__.py` | Qwen3.5-9B 클라이언트 (스트리밍/비스트리밍) |
| Pipeline | `src/pipeline.py` | RAG 오케스트레이터 + 트레이스 이벤트 |
| API | `src/api/__init__.py` | FastAPI + SSE 스트리밍 |
| Config | `src/config.py` | .env 기반 전역 설정 관리 |

### 3. 웹 UI 구현 (`frontend/index.html`)
- 다크 테마 채팅 인터페이스
- SSE 기반 실시간 토큰 스트리밍
- 마크다운 렌더링 (코드블록, 표, 불릿 등)
- 한국어 IME 처리 (compositionstart/compositionend)
- 바운싱 점 타이핑 인디케이터
- Pipeline Trace 패널 (오른쪽 접이식)

### 4. Pipeline Trace 패널 구현
LangSmith 유사 관찰가능성 기능:
- 파이프라인 9단계 실시간 추적
- 단계별 소요 시간(ms), 상태(done/hit/miss/streaming)
- 중간 결과 상세 표시 (쿼리, 스코어, 문서 미리보기)
- 쿼리별 그룹핑, 토탈 시간 표시
- boolean 값 시각적 구분 (✓ true 녹색, — false 회색)

## 발견한 이슈 & 해결

### 이슈 1: 한국어 IME 마지막 글자 잔류
- **증상**: Enter 키 입력 시 마지막 한 글자가 textarea에 남음
- **원인**: compositionend 이벤트 전에 keydown 처리
- **해결**: `isComposing` 플래그로 IME 조합 중에는 Enter 무시

### 이슈 2: LLM 응답이 세로로 출력
- **증상**: 토큰이 한 줄씩 세로로 표시됨
- **원인**: `white-space: pre-wrap` CSS + 토큰별 텍스트 삽입
- **해결**: pre-wrap 제거 + 마크다운 렌더링 함수 구현

### 이슈 3: 캐시된 응답 잘림
- **증상**: 캐시 히트 시 응답의 마지막 줄만 표시
- **원인**: SSE 멀티라인 data 필드가 분리됨
- **해결**: 모든 SSE data를 JSON.stringify()로 인코딩

### 이슈 4: 캐시 오탐 (False Positive)
- **증상**: 다른 질문인데 캐시 히트
- **원인**: similarity threshold 0.92가 너무 낮음
- **해결**: threshold를 0.95로 상향

### 이슈 5: Qwen3.5 Thinking 모드로 응답 지연
- **증상**: 일부 질문에 응답이 안 옴
- **원인**: Qwen3.5의 thinking 모드가 활성화되어 내부 추론에 시간 소모
- **해결**: `chat_template_kwargs: {"enable_thinking": False}` 추가

### 이슈 6: 스크래핑 데이터 품질 저하
- **증상**: "DeploymentandReplicaSet" 등 단어 붙음
- **원인**: HTML 태그 제거 시 공백 미삽입
- **해결**: chunker에 텍스트 정규화 단계 추가 + 기술 용어 보호 리스트

### 이슈 7: LLM이 문서 외 지식으로 답변
- **증상**: RAG 문서에 없는 내용을 LLM 자체 지식으로 답변
- **원인**: 시스템 프롬프트가 약해서 LLM의 parametric knowledge가 누출
- **해결**: 시스템 프롬프트 강화 ("사전 학습 지식 사용 금지" 규칙 추가)

### 이슈 8: 데이터 커버리지 부족
- **증상**: Deployment/StatefulSet, Probe 등 핵심 개념에 답변 불가
- **원인**: 스크래핑 데이터가 영어 + 내용 부실
- **해결**: 한국어 문서 직접 작성 + LLM 합성 문서 생성 파이프라인
