## 발견한 이슈 & 해결

### 이슈 1: 캐시 오탐 (Turn 3)
- **증상**: "리소스 제한 설정"이 "Pod 모니터링"과 유사하다고 캐시 히트
- **원인**: multilingual 모델이 한국어 의미를 더 잘 잡아서 오히려 유사도가 높게 나옴
- **해결**: threshold 0.92 → 0.95로 상향

### 이슈 2: "더 알려줘" 리라이트 누락 (Turn 5)
- **증상**: "더 알려줘"(6자)가 리라이트 안 됨
- **원인**: 짧은 질문 기준이 5자 미만이라 6자인 "더 알려줘"가 걸리지 않음
- **해결**: 기준을 8자 미만으로 상향

### 이슈 3: PDF 텍스트 공백 문제
- **증상**: PDF에서 추출한 텍스트에 불필요한 공백 포함 ("Pod 는", "Service 가")
- **원인**: PyPDF2의 한국어 텍스트 추출 특성
- **영향**: 검색에는 큰 영향 없음 (chunker 정규화에서 어느 정도 처리됨)
- **상태**: 모니터링 중, 심각하면 PDF 전용 후처리 추가 예정

### 이슈 4: 실용적 HOW-TO 질문 답변 부실
- **증상**: "Pod 리소스 제한 설정 방법"에 "제공된 문서에서 해당 정보를 찾지 못했습니다" 응답
- **원인**: 기존 문서가 개념 위주, YAML 예시/oc 명령어 부재
- **해결**: K8s Tasks 문서 18개 추가 스크래핑 + 한국어 실습 문서 3개 직접 작성

### 이슈 5: 교차 언어 검색 스코어 차이
- **증상**: 영어 원문(k8s_task_memory_resource.md)이 한국어 질문에 대해 top 10에도 못 들어옴
- **원인**: multilingual 모델이라도 동일 언어 매칭(0.76)이 교차 언어 매칭(0.4~0.5)보다 훨씬 높음
- **해결**: 핵심 실습 내용은 한국어 문서로 직접 작성, 영어 원문은 BM25 키워드 보조 역할
