### 3. 인접 청크 확장 (Adjacent Chunk Expansion) 추가

BM25 독립 검색으로 문서를 찾았지만, chunk 0 (개요) 만 검색 결과에 포함되어
LLM이 "구체적인 해결 절차를 찾지 못했다"고 답변하는 문제 발생.

**원인**: 실제 해결 방법은 chunk 1~11에 있으나, 검색 결과에 chunk 0만 포함됨.

**해결**: `Retriever._expand_adjacent_chunks()` 메서드 추가
- 검색된 청크의 앞뒤 ±2개 청크를 같은 문서에서 찾아서 텍스트 병합
- chunk_id 형식 `source::N`을 파싱하여 인접 청크 조회
- chunk 0이 잡히면 chunk 0~2의 텍스트가 합쳐져서 LLM에 전달

**결과**: 구체적인 진단 명령어(`oc describe pod`, `oc set image`)와 해결 절차까지 포함된 답변 생성

## 발견한 이슈 & 해결 과정

### 이슈: 문서를 찾았는데 LLM이 "못 찾았다"고 답변

- **증상**: BM25 독립 검색으로 `ocp-image-pull-troubleshooting-ko.md::0`이 1위로 올라왔지만, LLM은 "구체적인 해결 절차를 찾지 못했다"고 답변
- **원인**: chunk 0은 문서 개요(에러 종류 표)만 포함. 실제 진단/해결 명령어는 chunk 1~11에 있음. 검색 결과 5개 중 1개만 이 문서여서 context에 개요만 들어감
- **해결**: 인접 청크 확장 — 검색된 chunk의 앞뒤 ±2개를 자동으로 병합하여 context에 포함
- **효과**: chunk 0 → chunk 0+1+2 텍스트가 합쳐져서 "이미지 이름 오류" + "Pull Secret 미설정" 해결 방법까지 LLM에 전달

**근본 원인**: 기존 토크나이저가 한글을 글자 단위로 분리 ("상태" → "상","태")

1. **글자→단어 단위 분리**: `re.findall(r"[가-힣]+")` → `re.findall(r"[가-힣]{2,}")` 공백 기반
