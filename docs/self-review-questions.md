# 스스로에게 던져보는 질문지

---

## 프로젝트 주제

OCP(OpenShift Container Platform) 공식 문서 및 기술 자료를 기반으로 한 RAG(Retrieval-Augmented Generation) 챗봇.
사용자가 OCP 관련 질문을 하면, 관련 문서를 검색하여 LLM이 근거 있는 답변을 생성한다.

---

## 프로젝트 목표

| # | 목표 |
|---|------|
| G1 | RAG → LLM 파이프라인 + 웹 UI 구현 |
| G2 | Vector Index 설계 (IVF - K-Means, numpy 기반) |
| G3 | 멀티턴 대화 5턴 이상 지원 (세션 관리 + Query Rewriting) |
| G4 | LLM 연동 — Qwen3.5-9B (vLLM 서버) |
| G5 | Streaming 응답 (SSE 토큰 스트리밍) |

---

## 프로젝트 제약사항

| # | 제약 |
|---|------|
| C1 | RAG 프레임워크(LangChain, LlamaIndex 등)를 사용하지 않는다 |
| C2 | 벡터 DB(Chroma, FAISS, Qdrant 등)를 사용하지 않는다 |
| C3 | 핵심 로직(인덱싱, 검색, 리랭킹, 캐싱)은 외부 라이브러리에 위임하지 않는다 |
| C4 | 도구 성격의 패키지(numpy, torch, fastapi 등)는 허용한다 |
| C5 | 오프라인/에어갭 환경 배포를 고려한다 |

---

## 검토 질문

### A. 목표 달성 여부

A-1. [G1] RAG 파이프라인이 실제로 동작하는가? 질문을 입력하면 문서를 검색하고 답변이 생성되는가?
A-2. [G1] 웹 UI가 있는가? 브라우저에서 챗봇을 사용할 수 있는가?
A-3. [G2] IVF 벡터 인덱스가 구현되어 있는가? K-Means 클러스터링 코드가 실제로 존재하는가?
A-4. [G2] 벡터 인덱스가 전수검색(brute force) 대비 실제로 탐색 범위를 줄이는가?
A-5. [G3] 5턴 이상의 대화가 가능한가? 이전 대화 맥락을 기억하는가?
A-6. [G3] "그건 뭐야?" 같은 맥락의존 질문을 독립적 질문으로 변환하는가?
A-7. [G4] Qwen3.5-9B와 실제로 통신하는가? API 호출 코드가 존재하는가?
A-8. [G5] 답변이 한꺼번에 오는가, 아니면 토큰 단위로 스트리밍되는가?
A-9. [G5] 스트리밍 중에 Pipeline Trace(각 단계별 소요 시간)를 확인할 수 있는가?

### B. 제약 준수 여부

B-1. [C1] requirements.txt에 LangChain, LlamaIndex, Haystack 등이 포함되어 있지 않은가?
B-2. [C2] Chroma, FAISS, Qdrant, Pinecone 등 벡터 DB 의존성이 없는가?
B-3. [C3] 벡터 검색(IVF)의 클러스터링과 유사도 계산 코드가 프로젝트 내부에 있는가?
B-4. [C3] BM25 스코어링(TF, IDF, 길이정규화) 코드가 프로젝트 내부에 있는가?
B-5. [C3] 리랭킹 로직(Semantic + BM25 가중합)이 프로젝트 내부에 있는가?
B-6. [C3] 시맨틱 캐시(유사 질의 매칭)가 프로젝트 내부에 있는가?
B-7. [C4] numpy, torch, fastapi 같은 도구 패키지만 사용하고 있는가? 그 구분 기준은 명확한가?
B-8. [C5] 임베딩 모델을 로컬에서 로딩하는가, 외부 API를 호출하는가?
B-9. [C5] 인덱스가 파일(.npy)로 저장되는가, 외부 DB 서버가 필요한가?

### C. 검색 품질

C-1. "OCP에서 Pod 리소스 제한 설정 방법"을 물으면 관련 문서를 찾아 답변하는가?
C-2. "ImagePullBackOff 해결 방법"을 물으면 트러블슈팅 문서를 찾는가?
C-3. "새로 배포한 앱의 파드 상태가 Pending인 원인을 어떻게 파악하나요?" 같은 복잡한 한국어 질문에도 답변하는가?
C-4. "CLI에서 오픈시프트 클러스터에 로그인하는 방법"처럼 한영 혼용 질문에 대응하는가?
C-5. 검색 결과에 출처(문서명, 신뢰도)가 표시되는가?
C-6. 문서가 존재하지 않는 질문에 대해 "모른다"고 답변하는가, 환각(hallucination)을 내는가?

### D. 구조 및 확장성

D-1. 새로운 문서를 추가하려면 어떤 과정을 거치는가? (data/raw에 넣고 → 인덱스 재빌드?)
D-2. 문서 수가 현재(71개)에서 크게 늘어나면 IVF 클러스터 수(16)를 조정해야 하는가?
D-3. 임베딩 모델을 다른 것으로 교체하려면 어디를 수정해야 하는가?
D-4. LLM을 Qwen이 아닌 다른 모델로 바꾸려면 어디를 수정해야 하는가?
D-5. 각 모듈(vectorstore, retriever, cache 등)이 독립적으로 교체 가능한 구조인가?

### E. 추가 구현 사항

E-1. Dual-Path 검색(IVF + BM25)에서 두 경로의 역할 분담은 무엇인가?
E-2. Hybrid Reranking의 가중치(7:3)는 어떤 근거로 설정했는가?
E-3. 인접 청크 확장은 어떤 문제를 해결하기 위해 도입했는가?
E-4. 한국어 토크나이저 개선(단어 분리, 조사 제거, 동의어 매핑)이 실제로 검색 품질을 개선했는가? 전후 비교 결과는?
E-5. 2단계 캐싱(Embedding LRU + Semantic Cache)이 실제로 응답 속도를 개선하는가?
