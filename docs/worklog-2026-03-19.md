### 3. IVF n_probe 튜닝 (시도 → 원인 아니었음)

한국어 문서 추가 + 인덱스 재빌드 후에도 여전히 검색에 안 잡히는 문제 발생.
처음에는 IVF n_probe가 낮아서라고 판단하고 3→6으로 올렸지만, **brute force 검색과 결과가 동일하여 n_probe는 원인이 아니었음**.

**수정 (유지):**
- `src/config.py`: `IVF_N_PROBE` 설정 추가 (기본값 6) — 나쁠 건 없으니 유지
- `src/vectorstore/__init__.py`: 하드코딩된 `n_probe=3` → `IVF_N_PROBE` 참조로 변경

### 4. 근본 원인 진단: 임베딩 유사도 문제

**진단 결과:**
- `paraphrase-multilingual-MiniLM-L12-v2` 모델이 "이미지 풀 에러" 쿼리와 트러블슈팅 문서 청크 간 코사인 유사도를 **0.12~0.42**로만 생성
- 검색 결과 top-15 진입에는 **0.49+** 필요
- brute force (전수검색)에서도 동일 → IVF 클러스터링이 아닌 **임베딩 자체가 문제**

**구조적 문제:**
- BM25가 IVF 후보 15개 안에서만 reranking → 임베딩이 못 잡으면 BM25도 볼 기회 없음
- "이미지", "풀", "에러" 같은 정확한 키워드가 문서에 있는데도 검색 불가

### 5. 해결: BM25 독립 검색 경로 추가

BM25를 reranker에서 **독립 retriever로 승격**하여 해결.

**변경 전:** `IVF(15개) → BM25 rerank → top 5`
**변경 후:** `IVF(15개) + BM25 전체검색(10개) → 중복제거 + hybrid rerank → top 5`

**수정 파일:**
- `src/retriever/__init__.py`: `BM25Scorer`에 `index_corpus()`, `search_corpus()` 추가. `Retriever.retrieve()`에서 두 검색 결과 합침
- `src/api/__init__.py`: 서버 시작 시 `bm25.index_corpus()` 호출

**결과:**
- 트러블슈팅 문서가 검색 **1위**로 올라옴 (combined=0.50, keyword=1.0)
- 기존 질의에 대한 regression 없음

## 발견한 이슈 & 해결 과정

### 이슈 1: "이미지 풀 에러" 질의에 답변 불가
- **증상**: "배포 중 이미지 풀 에러가 나면 어떻게 해결하나요?" → "제공된 문서에서 찾을 수 없습니다"
- **1차 시도**: 영어 원문 스크래핑 (OCP/K8s 8개 URL) → 실패. 영어 문서는 한국어 질의와 임베딩 유사도가 낮아서 top 5에 안 잡힘
- **2차 시도**: 한국어 트러블슈팅 문서 직접 작성 (`ocp-image-pull-troubleshooting-ko.md`) → 인덱스 재빌드 후에도 실패
- **3차 시도**: 시맨틱 캐시 클리어 + n_probe 3→6 → 실패 (brute force와 결과 동일, n_probe는 원인 아님)
- **근본 원인**: 임베딩 모델이 해당 쿼리와 문서 간 유사도를 0.12~0.42로만 생성. top-15 진입에는 0.49+ 필요. BM25가 IVF 후보 안에서만 동작하여 키워드 매칭 기회 없음
- **최종 해결**: BM25를 reranker → **독립 retriever로 승격**. 전체 코퍼스에서 키워드 검색 후 IVF 결과와 합침 → 트러블슈팅 문서 **1위** 검색 성공

### 이슈 2: OCP Red Hat 문서 URL 3개 404 에러
- **증상**: `image-streams-using`, `troubleshooting-operating-system-issues`, `investigating-pod-issues` URL이 404
- **원인**: Red Hat 문서 사이트 URL 구조 변경 (4.17 버전)
- **상태**: 미해결. 올바른 URL 탐색 필요
