# Product Roadmap

이 문서는 `Play Book Studio`를 `OpenShift 4.20 플레이북 제품`으로 끌어올리기 위한 날짜별 목표를 고정한다.

원칙:
- 날짜가 지나도 해당 날짜 목표가 불합격이면 다음 단계로 넘어가지 않는다.
- 현재 우선순위는 `기본 챗봇 자격 회복 -> 시연 합격선 -> PlayBook 컨셉 강화` 순서다.
- 새 기능 추가보다 `실제 hard miss 해소`, `실제 멀티턴 안정화`, `실제 화면 상식 회복`을 우선한다.

## 2026-04-09

주제: `기본 챗봇 성능 정상화`

목표:
- 인사/가벼운 토크에 가볍게 응답
- `OCP가 뭐야`, `오픈시프트를 잘하려면 어떻게 해야돼` 같은 일반 질문에 답변
- no-answer 밑에 실패 유도 추천 질문 제거
- 시연용 예시 질문셋을 실제 통과 질문만으로 축소

통과 기준:
- `안녕`, `OCP가 뭐야`, `오픈시프트를 잘하려면 어떻게 해야돼`에 거절형 답변이 나오지 않음
- 첫 화면 예시 질문을 눌렀을 때 no-answer가 1회도 나오지 않음

## 2026-04-10

주제: `데모 합격선 확보`

목표:
- single-turn 시연 질문셋 확정
- multi-turn 5턴 시나리오 1개 확정
- 질문 -> 답변 -> 참조 -> 유저 매뉴얼북 -> 공식 원문 링크 흐름 고정
- 주제 전환 시 이전 질문 문맥이 새 답변을 오염시키지 않게 함
- follow-up일 때만 세션 문맥을 이어가고, 새 질문이면 주제를 리셋
- 세션 로그를 세션 ID별 파일로 남겨 복기 가능하게 유지

통과 기준:
- single-turn 6개 중 5개 이상 통과
- 서로 다른 주제 3회 전환 테스트에서 이전 질문 답변을 반복하지 않음
- 5턴 시나리오 1개를 끝까지 통과

## 2026-04-11

주제: `출력 품질과 UI 상식 회복`

목표:
- 채팅/참조/자료 패널 제목 타이포 규칙 통일
- 채팅 답변의 개행, 인라인 citation 위치, code block 렌더링 정리
- 전송/중지 버튼을 하나의 상태 시스템으로 정리
- 유저 매뉴얼북 내부에 `공식 매뉴얼 원문` 출처를 계속 확인 가능하게 노출

통과 기준:
- 답변 문장 끝 마침표와 citation 위치가 어색하지 않음
- 답변에 명령이 있으면 code block으로 보임
- `전송/중지/로딩` 상태가 중첩되지 않음

## 2026-04-12

주제: `PlayBook 데이터 1차 보강`

목표:
- 운영형/트러블슈팅 질문에서 비는 주제만 먼저 playbook 데이터로 보강
- 우선 후보:
  - Tekton PipelineRun webhook validation
  - 업그레이드 전 점검
  - 사용자 프로젝트 monitoring/alerting
  - Project Terminating
  - drain 후 Evicted/재스케줄링 실패

통과 기준:
- 위 주제 중 최소 3개에서 기존 no-answer가 playbook형 답변으로 바뀜

## 2026-04-13

주제: `PlayBook 완성 기준선`

목표:
- 공식 문서 + 운영형 playbook + 사용자 추가 문서의 3개 lane 정리
- 사용자 문서가 `유저 매뉴얼북`과 `RAG 코퍼스`로 동시에 변환되는 최소 경로 고정
- 운영형 질문에서 playbook 데이터가 우선적으로 잡히도록 정리
- 우측 참조 패널/유저 매뉴얼북/원문 출처까지 한 흐름으로 마감

통과 기준:
- `OpenShift 4.20 PlayBook`으로 설명 가능한 상태
- 질문/답변/근거/유저 매뉴얼북/원문 출처/운영형 playbook 답변이 한 흐름으로 연결됨

## 현재 1순위

지금 당장 가장 중요한 것은 `2026-04-09`와 `2026-04-10`을 통과시키는 것이다.

이유:
- 기본 챗봇 자격이 없으면 PlayBook 컨셉은 설득력을 잃는다.
- 내일 데모 합격선이 안 맞으면 이후 PlayBook 작업은 전부 설명력을 잃는다.
- 멀티턴이 오염되면 트러블슈팅 데이터가 들어와도 품질이 무너진다.
