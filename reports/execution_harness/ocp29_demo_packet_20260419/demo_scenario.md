# OCP 29 Gold Demo Scenario

## Demo Flow 1

1. Landing page 진입
2. `Launch Studio` 클릭
3. 챗봇에게 질문
4. 출력된 대답의 참조 문서 확인
5. 원본과 대조
6. `Gold Playbook` 이 공식 매뉴얼과 퀄리티 면에서 차이가 없음을 보여줌
7. 추가 질문 진행

## Multi-Turn Validation

- 추천 질문 3문항 중 하나를 눌러 추가 질문을 진행한다.
- 추천 질문을 누른 뒤 문서 확인까지 포함해 약 5턴 정도 반복한다.
- 이 구간에서 `multi-turn validation` 까지 함께 보여준다.

## Current Risk

- 지금 상태에서는 추천 질문 3문항 중 어느 것을 골라도 `자료없음` 으로 뜰 가능성이 있다.
- 이 리스크는 시연 품질에 직접 영향을 준다.

## User Library Demo

- 유저 데이터를 업로드해서 바로 `User Library` 화하는 흐름도 시연한다.
- 이 부분은 `Gold Playbook` 대비 퀄리티가 떨어질 것으로 예상한다.
- 이 부분은 내일까지 어떻게든 끝내야 한다.

## Library And Roadmap Segment

- `Playbook Library` 페이지로 들어간다.
- `Control Tower`
- `Repository 검색창`
- 위 두 요소를 보여주면서 로드맵 기획 발표를 진행한다.

## Closing Message For Stakeholders

- 내일까지 최종 완성을 낼 수 있다는 강한 자신감을 심어줘야 한다.
- 이를 위해 현재 파이프라인에서 문제인 부분을 밝힌다.
- 동시에 수정 계획을 이야기한다.

## Locked Narrative

- `29권 Gold` 는 현재 시연의 핵심 자산이다.
- 시연의 본론은 `quality parity + grounded navigation + multi-turn follow-up + user library path` 다.
- `113권 전권 완성` 은 오늘 시연의 본론이 아니다.
